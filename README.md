# Digital Terrain Model Viewer

Interactive terrain analysis and visualization of Digital Terrain Models (DTMs) in the browser.

[![Deploy to GitHub Pages](https://github.com/jwpro/digital-terrain-model-viewer/actions/workflows/deploy.yml/badge.svg)](https://github.com/jwpro/digital-terrain-model-viewer/actions/workflows/deploy.yml)

## Overview

This application demonstrates core terrain analysis algorithms — height map, hillshade, contour lines, slope, aspect, and curvature — running entirely client-side using OpenLayers and GeoTIFF data. No backend server or GIS software is required.

## Terrain Data Hosting

Large GeoTIFF files can exceed static host storage limits. You can host terrain data on object storage (Azure Blob, S3, Cloudflare R2, etc.) and point this app to that location.

Set an environment variable for production builds:

```bash
VITE_TERRAIN_DATA_BASE=https://your-data-host.example.com/digital-terrain-model-viewer/data/
```

If `VITE_TERRAIN_DATA_BASE` is not set, the app falls back to `${import.meta.env.BASE_URL}data/`.

## Terrain Analysis Features

- **Height Map** — Elevation colour mapping with configurable palette and vertical exaggeration
- **Hillshade** — Simulated sun illumination with azimuth, altitude, and shadow controls
- **Contour Lines** — Iso-height lines generated by the Marching Squares algorithm with labelling
- **Slope** — Terrain steepness from elevation gradient (degrees or percent)
- **Aspect** — Slope facing direction with 4/8/16-compass classification
- **Curvature** — Ridge and valley identification using second-order elevation derivatives

## Design System

All UI styling uses the `@jwrae/design-tokens` token system. No raw CSS values (colours, spacing, radii, shadows) appear anywhere in the codebase — every value is referenced through a CSS custom property.

## Algorithm Validation

Algorithm outputs are validated against QGIS reference results to ensure correctness.
